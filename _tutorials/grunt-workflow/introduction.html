Grunt is a command line JavaScript task runner. Essentially, what Grunt does, is automate tasks such as SCSS/SASS pre-processing, JavaScript linting, file minification, etc.

Grunt is a build tool for JavaScript, but can also be extended to perform tasks such as CSS & HTML processing and validation. It could be compared to other build tools such as Ant or Make, but for Web Developers & Designers rather than, for example, C++ developers.

Basically, Ben Alman's theory behind creating Grunt, was to create a tool that could perform specific sequences of task using one command, such as grunt build or grunt test.

Installing Grunt

Grunt is built using Node.JS and is installable using the 'Node Package Manager'. We're not going to go through installing Node.JS in this tutorial, as there are many options for installation and the process is beyond the scope of this guide.

You will want to install Grunt globally, as well as the CLI tool for Grunt, so just issue the following commands to get Grunt setup on your machine.

npm install -g grunt
npm install -g grunt-cli

During the install process, NPM will install all of the required dependencies required to run Grunt. Once the install process finishes, you will have Grunt installed and ready to use on your machine. Make sure you don't leave out the -g in the commands or you'll run into issues later on.
